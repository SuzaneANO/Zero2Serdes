# Project Environment & Xschem Setup Guide

This document summarizes the environment configuration and troubleshooting steps taken to establish a stable workflow for the OpenLane2 / Sky130 project.

## 1. Git Tag Management

To manage project milestones and versions:

- **List available tags:** `git tag`
    
- **Filter tags:** `git tag -l "v1.*"`
    
- **Checkout a specific tag (Read-only):** `git checkout <tag_name>`
    
- **Create a branch from a tag (To work on it):** `git checkout -b <new_branch_name> <tag_name>`
    

## 2. Persistent Environments with `direnv`

To ensure `xschem` and OpenLane tools are available in all subdirectories without manually re-entering the Nix shell.

### Installation & Shell Hook

1. **Enable the Hook:** Add this to your `~/.bashrc`:
    
    ```
    eval "$(nix run nixpkgs#direnv -- hook bash)"
    ```
    
2. **Reload Shell:** `source ~/.bashrc`
    

### Project Activation

In your project root directory:

1. **Create config:** `echo "use flake" > .envrc` (or `use nix`)
    
2. **Authorize:** `direnv allow`
    
3. **Result:** Navigating into any subfolder now automatically loads the required tools.
    

## 3. Xschem & Sky130 PDK Fixes

The primary issue involved Xschem failing to start because the `xschemrc` was pointing to a missing or deeply nested Volare PDK path.

### Finding the PDK Path

If the PDK is moved or nested, use `find` to locate the correct tech file:

```
find .volare -name xschemrc
```

### Correcting `work/xschemrc`

Update your local `xschemrc` to use an **absolute path** to the Sky130 tech file.

**Avoid relative paths** like `../tech/...` if your folder structure is non-standard.

**Recommended `xschemrc` content:**

```
# Source the actual PDK tech file found via 'find'
source /mnt/c/Users/AdamSbane/Downloads/OpenLane/openlane2/.volare/volare/sky130/versions/.../sky130A/libs.tech/xschem/xschemrc

# Add your local project designs to the library
append XSCHEM_LIBRARY_PATH :../design
```

### Launching without Errors

If Xschem complains about `Unable to open file ... top.sch`, bypass the default auto-load by specifying your schematic file directly:

```
xschem -s ../design/RPLY_EX0_SKY130NM/RPLY_EX0.sch
```

## 4. Troubleshooting Checklist

- **Command not found?** Run `direnv allow` in the root.
    
- **Red boxes in Xschem?** Check if `XSCHEM_LIBRARY_PATH` in `xschemrc` points to the `libs.ref` folder in the PDK.
    
- **Cat: file not found?** Ensure the `-o xsch/` flag is used in your netlisting command to direct output to the correct folder.
